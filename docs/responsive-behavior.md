# Railora One Responsive Behavior

## Desktop

Desktop uses a persistent left navigation rail, sticky top bar, command palette trigger, workspace/environment controls, privacy mode, notification control, and user menu. Operations pages may use two-column layouts and sticky side panels.

## Tablet

Tablet views collapse dense grids into two columns where possible. Tables stay inside scroll containers. The experience should not be a squeezed desktop layout.

## Mobile

Mobile uses a five-item bottom navigation: Home, Payments, Move, Insights, and More. More opens a bottom menu for secondary routes. Primary financial actions remain reachable with one thumb, and fixed navigation must not cover submit buttons or review actions.

## Overflow Contract

No route should create page-level horizontal scroll at supported viewport widths. Wide tables, ledger previews, reconciliation views, and API logs must scroll inside their own containers.
